ShopIQ

Once a customer signs up, they will need to integrate their website with IQLECT platform to start analysing data that is flowing from both visitor’s (clickstream) perspective and an e-commerce standpoint.

Browser-based Analytics JS library Integration

Tracking visitor analytics and behaviour is essential in order to provide accurate and visitor-specific results and prediction. IQLECT analyses visitor events, such as product clicks, products added to cart, orders, etc. These events are tracked using browser cookies. The conventional way to integrating IQLECT Analytics is a simple two-step process:

Including IQLECT js library script

Your first step is to insert a JavaScript code within all the HTML pages in your website. This JavaScript code is responsible for pushing the visitor events to IQLECT.

Notes:

  • The JavaScript code will not affect the performance of the page as it is loaded asynchronously.
  • This code could be inserted anywhere in the page (header, body, footer) and need not be included in the head section of the HTML.

Tracking Events:

When visitors browse through your store, the integrated trackers log everything visitors do - the products they visit, orders, even the various store properties they interact with. The trackers are unique tracking codes that must be configured onto the store properties that yield an interaction. We call this interaction as an "event", for example, on the "Add to Cart" button

With each event there are certain additional details that needs to be sent along with the event. For example when a product will be added to the cart when you click on the ‘ add to cart’ button. Here, you can call the Iqlect method to send ‘cart information’.

Like this :

var cartObj = { 
    'customer_id': 1234,
    'cart_token': 1111, 
    'total_items_price': 1140.00, 
    'subtotal_price': 1154.20, 
    'total_price': 1154.20, 
    'discount_code': '' 
    .... }

Upload Customer Customer CSV

A list of your customers with details like customerId, emailId, first name, last name etc... This data will be helpful for when we are tracking a customer online and want to relate the activities with customer. Along with this its important from lead scoring dashboard standpoint as well.

Format : Sample CSV

Upload Product Catalog CSV

A list of product in your current catalog to the platform. This important for us to track and identify products and activities which each customer is performing with each PDP page. The productId, pricing and other details are further used as part of propensity scoring.

Format : Sample CSV
