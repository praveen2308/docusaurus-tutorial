ShopIQ Analytics Integration

Tracking visitor analytics and behaviour is essential in order to provide accurate and visitor-specific reports and insights. ShopIQ analyzes visitor events, such as product clicks, products added to cart, orders, etc. These events are tracked using browser cookies. With this information, a profile is built for every visitor, based on his/her affinity to different categories, brands, or prices.

This clickstream data is also used to determine the effectiveness of the site as a channel-to-market. It also takes into account with what pages the customer lingers on, the products that a customer is most likely to purchase or is abandoning, whether or not the customer belongs to a ​loyalty program​ and uses a coupon code and their preferred mode of payment, etc.

How to Integrate?

Types of Analytics Integration:

  • Browser-based Analytics Integration
  • REST API-based Analytics Integration (Comming Soon)

We do recommend that the customer follow Browser-based Analytics Integration, where in customer does the integration by including ShopIQ javascript library. ShopIQ javascript library performs all the operations without the customer investing integration effort. But in case of mobile SDK, Amazon Alexa or any kind of non-browser based ShopIQ integration, customers have to use RESTful API-based analytics integration.

The conventional way to integrating ShopIQ Analytics is a simple two-step process:

  1. Inserting a JavaScript code.
  2. Tracking events.

1. Inserting the JavaScript Code

Your first step is to insert a JavaScript code within all the HTML pages in your website. This JavaScript code is responsible for pushing the visitor events to ShopIQ.

Notes:

  • The JavaScript code will not affect the performance of the page as it is loaded asynchronously.
  • This code could be inserted anywhere in the page (header, body, footer) and need not be included in the head section of the HTML.

Once you login to the platform you can go to the "App Settings" section from the left hand sidebar. After that just click on the JS section you will see a script. Which needs to be added to your website.

Sample Script :

<script type="text/javascript" src="https://objectstorage.eu-frankfurt-1.oraclecloud.com/n/lrgvv9ratlcw/b/iqlect-send-client/o/js-clients/tags/tENSFY7tAAAA.js"></script>
<script type="text/javascript">
  var _iqparams = _iqparams || {};
  var _iqexparams = _iqexparams || {};
  var _iqevents = _iqevents || [];
  (function() {
    var u="http://130.61.29.96:8080/";
    var s3_u="https://objectstorage.eu-frankfurt-1.oraclecloud.com/n/lrgvv9ratlcw/b/ShopIQ-send-client/o/js-clients/v3/";
    _iqparams.endpoint = u+'iqtracker';
    _iqparams['streams'] = {PRODUCT: "product_ENSFY7tAAAA",CUSTOMERS: "customers_ENSFY7tAAAA",PAGE_EVENTS: "page_events_ENSFY7tAAAA",ORDERS: "orders_ENSFY7tAAAA",PRODUCT_VIEWED: "product_viewed_ENSFY7tAAAA",VISITOR: "visitor_ENSFY7tAAAA",ORDER_PRODUCT_DETAILS: "order_product_details_ENSFY7tAAAA"};
    _iqparams['accountId'] = 'EKXSvvLAAAE';
    _iqparams['schema-name'] = 'GlobalSchema_EKXSvvLAAAE';
    _iqparams['stream-name'] = 'product_ENSFY7tAAAA';
    var d=document, g=d.createElement('script'), s=d.getElementsByTagName('script')[0];
    g.type='text/javascript'; g.async=true; g.defer=true; g.src=s3_u+'ShopIQ.min.js'; s.parentNode.insertBefore(g,s);
  })();
</script>
<noscript><p><img src="https://csuccessportal.ShopIQ.com/iqtracker?idsite=" style="border:0;" alt="" /></p></noscript>

This JavaScript snippet automatically:

  • Tracks new visitors and makes their profiles (files) using browser cookies. This information is stored within the browser cookie having the name ‘ShopIQ.sessionId’ .
  • Tracks and pushes the events to ShopIQ.

2. Tracking events.

The actions a visitor takes on your ecommerce store are known as Events. Please follow the below steps for analysis to take place without any issues. There are few parameters which are not mandatory, so in case, you don’t have them then that can be left blank.

Events :

  1. Page Type
  2. Product Details
  3. Add to cart
  4. Checkout
  5. Orders
  6. Custom Events

1. Page Type

We need to set pageType variable in all pages, depending on the page type.

JavaScript
_iqexparams.pageType = "<PAGE TYPE>";

<PAGE TYPE>: This could be any of the below mentioned values. The pageType variable is based on the current page. Just assign the values to this variable based on the page the customer is on.

Different Types of PageTypes

Ecommerce PagePageType Value
Home Pagehome
Product Listing Pagelisting
Product Description Pageproductpage

2. Product Details

The below code should be added on all product details pages. And in your js code make sure the below values are passed correctly.

JavaScript
_iqexparams.product_id = "<PRODUCT ID>"; 
_iqexparams.product_name = "<PRODUCT NAME>"; 
_iqexparams.brand_name = "<BRAND>"; 
_iqexparams.category = "<CATEGORY>"; 
_iqexparams.price_per_unit = "<PRICE>";

The values that needs to be passed :

<PRODUCT ID>: This would be the productId of the product. It should be unique across the whole catalog. <PRODUCT NAME>: Title of the product.
<BRAND>: The name of the brand this product is associated with.
<CATEGORY>: The category this product is associated with. It would be the primary Category Name. The same name should be part of the catalog CSV uplodaed to the platform.
<PRICE>: The selling price per unit of this product.

3. Add To Cart

The below code should be executed when a user clicks on "Add To Cart" button. Here, you can call the ShopIQ method to send ‘cart information’.

JavaScript
var cartObj = { 
      customer_id: '<CUSTOMER ID>',
      cart_token: '<CART TOKEN>', 
      discount_code: '<DISCOUNT CODE>', 
      shipping_charges: <SHIPPING CHARGES>, 
      products:[
          {
            product_id: '<PRODUCT ID>'
            price: <PRICE>, 
            discount: <DISCOUNT VALUE>, 
            quantity: <QUANTITY>
          }, 
          {
            // 2nd Product Details
          } 
      ],
      total_items_price: <TOTAL PRICE>, 
      subtotal_price: <SUB TOTAL PRICE>, 
      total_price: <TOTAL PRICE>, 
      total_discount: <TOTAL DISCOUNT>, 
} 
ShopIQ.add_to_cart(cartObj);

<CUSTOMER ID>: The unique ID to identify customer. This ID should be similar to the ID uploaded in the customer CSV uploaded to the platform. empty in case not logged in. And in case user has logged in please pass the userId.
<CART TOKEN>: The unique cart token generated by your platform.
<DISCOUNT CODE>: In case a discount code was offered for this product. If not you can send empty value as well.
<SHIPPING CHARGES>: The Shipping Charges applied for this product.
<TOTAL PRICE>: The total price value of the cart. Price of the product * Quantity - discount price (After removing discount).
<SUB TOTAL PRICE>: The sub total price value of the cart.
<QUANTITY>: Quantity for each product.
<DISCOUNT VALUE>: Discount value for each product.
<PRICE>: Final selling price for including quantity, price per product and discount for each product.
<TOTAL DISCOUNT>: Discount for each product * Quantity.

4. Checkout

After clicking on checkout/proceed to checkout button, user will land on checkout page. Here, we need to call checkout method(which will send the cart information). To send the checkout event, we need to follow the event similar to “add to cart”, only the method name changes as mentioned below.

JavaScript

      customer_id: '<CUSTOMER ID>',
      cart_token: '<CART TOKEN>', 
      total_items_price: <TOTAL PRICE>, 
      subtotal_price: <SUB TOTAL PRICE>, 
      total_price: <TOTAL PRICE>, 
      total_discount: <TOTAL DISCOUNT>, 
      discount_code: '<DISCOUNT CODE>', 
      products :[
            {
              product_id: '<PRODUCT ID>'
              price: <PRICE>, 
              discount: <DISCOUNT VALUE>, 
              quantity: <QUANTITY>
            }, 
            {
              product_id: '<PRODUCT ID>'
              price: <PRICE>, 
              discount: <DISCOUNT VALUE>, 
              quantity: <QUANTITY>
            } 
        ]
} 
ShopIQ.checkout(​checkoutObj​);

<CUSTOMER ID>: The unique ID to identify customer. This ID should be similar to the ID uploaded in the customer CSV uploaded to the platform. empty in case not logged in. And in case user has logged in please pass the userId.
<CART TOKEN>: The unique cart token generated by your platform.
<DISCOUNT CODE>: In case a discount code was offered for this product. If not you can send empty value as well.
<SHIPPING CHARGES>: The Shipping Charges applied for this product.
<TOTAL PRICE>: The total price value of the cart. Price of the product * Quantity - discount price (After removing discount).
<SUB TOTAL PRICE>: The sub total price value of the cart. //--- Need info ----// <QUANTITY>: Total quantity for each product.
<DISCOUNT VALUE>: Discount value for each product.
<PRICE>: Final selling price for including quantity, price per product and discount for each product.
<TOTAL DISCOUNT>: Discount for each product * Quantity.

5. Order

After completing the payment and a successful order, we need to send order details to ShopIQ using ​ShopIQ.order_placed(orderObj)​. This method needs to be called only once. If you call the method on refresh of order success page, it will make one more entry in ShopIQ db and then it will be twice the same order.

JavaScript
var orderObj = { 
      customer_id: '<CUSTOMER ID>',
      cart_token: '<CART TOKEN>', 
      order_id:'<ORDER ID>', 
      email: '<CUSTOMER EMAIL ID>', 
      billing_city: '<CUSTOMER BILLING CITY>', 
      billing_country: '<CUSTOMER BILLING COUNTRY>', 
      billing_state: '<CUSTOMER BILLING STATE>', 
      billing_postcode: <CUSTOMER BILLING POSTAL CODE>, 
      shipping_city: '<CUSTOMER SHIPPING CITY NAME>', 
      shipping_country:  '<CUSTOMER SHIPPING COUNTRY>', 
      shipping_state: '<CUSTOMER SHIPPING STATE>', 
      shipping_postcode: <CUSTOMER SHIPPING POSTAL CODE>, 
      payment_method: '<PAYMENT METHOD>', 
      gateway: '<PAYMENT GATEWAY>', 
      currency_code: '<CURRENCY CODE>', 
      total_items_price: <TOTAL ITEM PRICE>, 
      subtotal_price: <SUB TOTAL PRICE>, 
      total_price: <TOTAL PRICE>, 
      discount_code: '<DISCOUNT CODE>', 
      total_discount: <TOTAL DISCOUNT>, 
      shipping_charges: <SHIPPING CHARGES>, 
      products:[ 
          {
              product_id: '<PRODUCT ID>'
              price: <PRICE>, 
              discount: <DISCOUNT VALUE>, 
              quantity: <QUANTITY>
        },
        {
              product_id: '<PRODUCT ID>'
              price: <PRICE>, 
              discount: <DISCOUNT VALUE>, 
              quantity: <QUANTITY>
        }
    ]
} 
ShopIQ.order_placed(orderObj);

<CUSTOMER ID>: The unique ID to identify customer. This ID should be similar to the ID uploaded in the customer CSV uploaded to the platform. empty in case not logged in. And in case user has logged in please pass the userId.
<CART TOKEN>: The unique cart token generated by your platform.
<ORDER ID>: The unique order ID generated by the store.
<CUSTOMER EMAIL ID>: The unique emailId used by customer to create an account in the store.
<CUSTOMER BILLING CITY>: The name of the city from where the customer is making the order.
<CUSTOMER BILLING STATE>: The name of the state from where the customer is making the order.
<CUSTOMER BILLING COUNTRY>: The name of the country from where the customer is making the order.
<CUSTOMER SHIPPING CITY>: The name of the city from where the customer is making the order.
<CUSTOMER SHIPPING STATE>: The name of the state from where the customer is making the order.
<CUSTOMER SHIPPING COUNTRY>: The name of the city from where the customer is making the order.
<PAYMENT METHOD>: The payment method used for making this order. Eg. Net Banking, Credit Card etc.
<PAYMENT GATEWAY>: The payment gateway used for billing. Eg. Paypal, Paytm etc.
<CURRENCY CODE>: The currency code used for billing. Eg. INR for Indian Ruppes, USD for US Dollars etc. <DISCOUNT CODE>: In case a discount code was offered for this product. If not you can send empty value as well.
<SHIPPING CHARGES>: The Shipping Charges applied for this product.
<TOTAL PRICE>: The total price value of the cart. Price of the product * Quantity - discount price (After removing discount).
<SUB TOTAL PRICE>: The sub total price value of the cart. //--- Need info ----//
<QUANTITY>: Total quantity for each product.
<DISCOUNT VALUE>: Discount value for each product.
<PRICE>: Final selling price for including quantity, price per product and discount for each product.
<TOTAL DISCOUNT>: Discount for each product * Quantity.
